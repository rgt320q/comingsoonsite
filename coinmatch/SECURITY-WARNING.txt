ORTA SEVİYE GÜVENLİK ⚠️

✅ Ne Korunuyor:
  • Firebase Security Rules aktif (sadece giriş yapanlar)
  • Authentication zorunlu (Google hesabı)
  • Veri okuma/yazma kuralları var
  • Firebase otomatik rate limiting

⚠️ Risk:
  • API anahtarları görünür (browser'da)
  • Herhangi bir domain'den kullanılabilir (şu anda)

2️⃣ AUTHORIZED DOMAINS (Giriş Koruması)
   
   Adımlar:
   a) Firebase Console → Authentication
   b) Settings → Authorized domains
   c) Sadece şunları bırakın:
      • sizin-domain.com
      • localhost (sadece test için)
   
   d) Diğerlerini silin!

3️⃣ FIRESTORE RULES KONTROLÜ (Veri Koruması)
   
   Firebase Console → Firestore Database → Rules
   
   Şu kurallar aktif mi kontrol edin:
   - Okuma: auth != null (sadece giriş yapanlar)
   - Yazma: auth != null && (koşullar)

🛡️ İLERİ SEVİYE KORUMA (Opsiyonel Ama Önerilen)

4️⃣ FIREBASE APP CHECK (Bot Koruması)
   
   Firebase Console → App Check
   • reCAPTCHA v3 aktif edin
   • Bot saldırılarını engeller


5️⃣ CONTENT SECURITY POLICY (CSP)
   
   index.html dosyasına eklenmiş CSP header
   • Sadece belirlenen kaynaklardan script yükler
   • XSS saldırılarını önler


6️⃣ MONITORING (İzleme)
   
   Firebase Console → Analytics
   • Anormal aktivite takibi
   • Kullanım limitleri kontrolü

❓ SIKÇA SORULAN SORULAR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

S: API key görünse bile güvenli mi?
C: EVET, domain restriction yaparsanız. Firebase client-side 
   API key kullanımı için tasarlanmıştır. Asıl koruma domain 
   restriction ve security rules'dadır.

S: Biri API key'i kopyalayıp kullanabilir mi?
C: Domain restriction varsa HAYIR. Farklı domain'den 
   kullanamazlar. Security Rules varsa veriyi okuyamazlar.

S: API key'i gizli tutmalı mıyım?
C: Mümkünse evet, ama görünse de sorun değil. Önemli olan 
   Firebase Console'daki koruma ayarları.

S: Hacklenmeme riskim nedir?
C: Domain restriction + Security Rules ile: %5
   Bunlar olmadan: %30
   App Check ile: %1

S: Firebase ücretsiz mi?
C: Evet, günlük limitler dahilinde. Limitler:
   • 50,000 okuma
   • 20,000 yazma
   • 10GB transfer

S: Limit aşarsam ne olur?
C: Firebase servisler durur veya ücretli plana geçersiniz.
   Monitoring ile takip yapın.

✅ GÜVENLİK KONTROL LİSTESİ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Sunucuya yüklemeden ÖNCE:
□ API key domain restriction ayarlandı
□ Authorized domains güncellendi
□ Firebase Security Rules kontrol edildi
□ Test domain'leri kaldırıldı (production'da)

Sunucuya yükledikten SONRA:
□ Gerçek domain ile test edildi
□ Google giriş çalışıyor
□ Veri okuma/yazma çalışıyor
□ Başka domain'den test edildi (erişememeli!)

İleri seviye:
□ App Check aktif edildi
□ CSP header eklendi
□ Firebase Analytics aktif
□ Monitoring kuruldu

🚨 ACİL DURUM: API Key Sızdıysa

5. Domain restriction'ı yeni key'e uygulayın
